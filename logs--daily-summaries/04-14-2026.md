# 04-14-2026

## Summary

Built the full terminal typesetting engine (Phases 0–3 of the Pretext integration roadmap). Discovered Pretext.js is browser-only (Canvas dependency) and designed a terminal-native alternative adopting its two-phase architecture. Expanded the roadmap from 16 to 72 items across 8 phases. Built 10 motion/animation primitives, 18 visualization primitives, and integrated them into the TUI views.

## What was done

### Roadmap
- Rewrote `docs--roadmap/pretext-integration.md` — expanded from 4 phases / 16 items to 8 phases / 72 items
- Added Phase 0 (terminal-native layout engine), Phase 5 (composition/panels), Phase 6 (interaction), Phase 7 (accessibility)
- Noted critical finding: Pretext.js requires browser Canvas `measureText()` — not usable in terminal context

### Phase 0 — Core engine (`src--tui-and-renderer/typeset.ts`, ~530 lines)
- `charWidth()` — Unicode-aware column width (CJK, emoji, combining marks, zero-width)
- `stringWidth()` — ANSI-aware visible width with surrogate pair handling
- `prepare()` — ANSI-aware text segmenter with whitespace modes (normal / pre / pre-wrap)
- `layout()` — pure-arithmetic line breaking over cached segments
- `materialize()` / `materializeToStrings()` — lazy string building from line ranges
- `walkLineRanges()` / `measureLineStats()` / `measureNaturalWidth()` — measurement without allocation
- `layoutNextLineRange()` — variable-width line iteration for text around panels
- `prepareRichInline()` — mixed-style inline text convenience
- `balancedWidth()` — binary search for optimal width (prevents orphan words)

### Phase 1 — Composition utilities (`src--tui-and-renderer/typeset-compose.ts`, ~300 lines)
- `truncatePath()` — middle-out path truncation preserving first + last segments
- `truncateEnd()` — ANSI-aware end truncation with ellipsis
- `columns()` — flex-based columnar layout with alignment and truncation
- `justifyLine()` / `justifiedLayout()` — full-justification with ANSI awareness
- `box()` — box model with padding, border (single/double/rounded/heavy), margin, alignment
- `hangingIndent()` — first-line full-width, continuations indented
- `breakpoint()` / `breakpointName()` — responsive breakpoints (compact/standard/wide)

### Reactive integration (`src--tui-and-renderer/typeset-reactive.ts`, ~110 lines)
- `createLayout()` — SolidJS memo chain (prepare → layout → materialize)
- `createBalancedLayout()` — auto-balancing variant
- `createLineStats()` — lightweight reactive measurement
