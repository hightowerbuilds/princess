# Landing Page Context

The product is a local-first prompt inbox for people who work with coding agents. The landing page should explain why durable prompt storage matters, show that Princess stores ordinary files on disk, and make the CLI/TUI workflow feel approachable.

Primary audience:

- Developers who already use terminal tools.
- Product/design operators who ask agents to create reusable prompts.
- Teams that want prompt handoffs to survive beyond a chat transcript.

Important product claims:

- Prompts live as Markdown files or HTML prompt workspaces.
- Agents can save prompts directly into the inbox.
- HTML workspaces can carry structured sections, local sources, tables, and asset metadata.
- The app is local-only and avoids database/sync complexity.

Tone:

- Practical, precise, and calm.
- No exaggerated AI hype.
- Show the workflow quickly.
