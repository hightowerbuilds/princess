# UI Review Notes

The screenshot represents a dashboard-style prompt inbox with a sidebar, a main list, and a detail panel.

Review goals:

- Check whether the selected prompt is visually obvious.
- Verify that read-only HTML workspaces are distinguishable from editable Markdown prompts.
- Evaluate whether the search field and copy action are discoverable.
- Identify spacing or hierarchy problems that would slow repeated daily use.

Known constraints:

- The app is terminal-first, so recommendations should focus on layout, labels, state cues, and information density.
- Do not recommend cloud sync, collaboration, or database features for this review.
- Prefer changes that preserve the current local-file mental model.
